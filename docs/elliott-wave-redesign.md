# 엘리엇 파동 검출 전면 재설계

## 문제 (기존 구현의 근본 결함)
현 `elliott.py`는 전 구간에서 **신뢰도 최고 5파 하나만** 골라 라벨한다. 그 결과:
1. **과거 한 스냅샷 고정** — 과거의 깨끗한 5파가 뽑히고 현재 구조와 무관
2. **상승 5파 편중** — 하락 조정 3파(ABC)를 카운팅 안 함
3. **프랙탈 없음** — 장기 파동 내 단기 파동(등급)이 없음
4. **현재 위치 판단 불가** — "지금 몇 파"가 안 나옴

## 리서치 결론 (권위 소스 교차검증)
- **3대 하드룰**(절대 게이트): R1 2파는 1파 시작 비침범, R2 3파 최단 아님, R3 4파가 1파 영역 비중첩. 상승/하락 미러.
- **피보나치는 소프트**(점수용, 게이트 아님): 2파 0.5~0.618, 3파 1.618, 4파 0.382, 5파≈1파. ±5% 존.
- **아키텍처**: 재귀 ZigZag(다중 등급, 엄격한 nesting) + 전 구간을 임펄스+조정 교대 체인으로 파싱(고립된 5파 사냥 금지) + 프랙탈 교차검증(추진=하위 5스윙, 조정=3스윙).
- **현재 위치**: 최고점수 체인의 종단 피벗 = 현재 파동. 무효화가격(하드룰 유도) 함께 제시. 우측 끝은 확률로(단정 금지).
- **정직성**: 우측 끝 모호성·리페인트·복합조정은 라벨 유보가 정답(elastic relabeling 회피).

## 확정 설계 (실측 검증 완료, 35종목 데이터셋)

### 1. 다중 등급 피벗 (재귀 ZigZag)
- minor: 원가격 ZigZag 5% → 세부 파동
- major: minor 피벗 위에 재귀 ZigZag(~13% 스윙) → 큰 파동. **major ⊆ minor 보장**(삼성 21⊆53 검증).

### 2. 전 구간 체인 파싱 (DP)
각 등급 피벗열을 좌→우 DP로 임펄스(5레그)·조정(ABC 3레그) 세그먼트 체인으로 분할.
- 하드룰 통과 + fib_score≥0.3 세그먼트만, coverage+confidence 최대화.
- gap(skip) 허용(작은 페널티) — 안 맞는 구간은 라벨 유보(정직).
- 실측: minor 57% 커버리지, ~9.5 세그먼트/종목(전 구간 흐름), major ~1.6.

### 3. 현재 위치 + 무효화
마지막 세그먼트 + 이후 진행 레그 수로 판정:
- 임펄스 뒤: 5파완성/A/B/C파. 무효화=5파 극점.
- 조정 뒤: 완료/1/2/3파. 무효화=조정 시작.
- 진행 레그>3(복합/연장): "복합 구간(라벨 유보)" — 억지 카운트 안 함.

### 4. 출력 (스키마 확장)
- `segments`: [{start_date, end_date, kind(impulse|correction), degree(major|minor), labels, direction, confidence}]
- `current_position`: 사람이 읽는 현재 파동 문구
- `invalidation_price`: 카운트 무효 경계(있으면)
- 기존 `pivots`(등급별)·`labeled`·`note` 유지(하위호환).

## 프론트
- 오버레이: minor 세그먼트를 임펄스=실선/조정=점선, 등급별 색/두께. 라벨(1~5, A~C) 마커.
- 현재 위치 배지 + 무효화가격(TrendPanel).
- major 파동은 굵게, minor 파동은 가늘게 겹쳐 프랙탈 노출.

## 정직성 원칙 (사용자 오해 방지)
- 라벨 안 붙는 구간은 그대로 두기(스윙 점선). 전 구간 억지 라벨 금지.
- 현재 위치는 "추정 + 무효화가격"으로, 확정 아님을 문구로.
