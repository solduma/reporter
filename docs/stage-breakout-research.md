# 와인스타인 국면 판별 — 돌파·이탈·시장구조 반영 리서치

작성: 2026-07-14. 대상: `api/app/domain/stage.py` (와인스타인 4국면 판별기).

## 요약(TL;DR)

지적이 정확하다. 현재 판별기는 **가격 vs MA·MA기울기 + shape(회귀·ER·곡률)** 이 뼈대이고,
Weinstein 방법론의 **핵심 트리거인 "거래 레인지(박스권) 돌파/이탈 + 거래량 폭발"** 은 사실상
빠져 있다. 구체적으로:

1. **밴드(segments)에는 돌파·거래량·변동성이 아예 안 들어간다.** `segments()` 가
   `classify(closes)` 를 **종가만** 넘겨 호출해서, breakout·volume_signal·volatility·channel_pos
   가 전부 기본값(none/neutral/normal/None)으로 죽는다. 즉 화면 배경밴드와 스크리너 trend 배치의
   국면은 **거래량·돌파를 한 번도 못 본다.** (`/analysis` 배지만 OHLCV 를 넘겨 예외)
2. **Donchian `channel_pos` 는 계산만 하고 `_decide` 에 안 넘긴다.** 완전한 dead code.
3. **스윙 구조(LL·HL·LH·HH), 박스권(단기 고점끼리·저점끼리), 갭** 개념이 코드에 없다.
   Weinstein·Dow 이론의 1차 구조 신호가 통째로 빠져 있다.
4. **상대강도(Mansfield RS)** 는 별도 모듈에 있으나 국면 판별에 안 쓰인다. Weinstein 은
   Stage 2 진입 확인에 RS 상향 돌파를 필수로 본다.

결과적으로 "Stage 1→2 조짐 타점", "Stage 2 유지 확인 타점", "Stage 4 이탈 매도 타점" 을
**정밀하게** 잡기 위한 1차 재료(레인지·거래량·구조)가 부족하다.

## 1. 현재 판별기가 보는 것 / 못 보는 것

| 신호 | 계산됨? | `classify()` 반영? | `segments()`(밴드) 반영? |
|---|---|---|---|
| 가격 vs MA (above/near/below) | ✓ | ✓ | ✓ |
| MA 기울기 (rising/flat/falling) | ✓ | ✓ | ✓ |
| 로그회귀 기울기·R² | ✓ | ✓ | ✓ |
| Efficiency Ratio | ✓ | ✓ | ✓ |
| 곡률(U자/역U자) | ✓ | ✓ | ✓ |
| **거래량 축적/분산 + OBV** | ✓ | ✓(analysis만) | ✗ (종가만 넘김) |
| **변동성 레짐(ATR 수축/확장)** | ✓ | ✓(analysis만) | ✗ |
| **Donchian 돌파(up/down)** | ✓ | ✓(analysis만) | ✗ |
| **Donchian 채널 위치(0~100)** | ✓ | **✗ (_decide 미전달)** | ✗ |
| **스윙 구조 LL/HL/LH/HH** | ✗ | ✗ | ✗ |
| **박스권(수평 지지/저항)** | ✗ | ✗ | ✗ |
| **거래량 폭발 갭/클라이맥스** | ✗ | ✗ | ✗ |
| **상대강도(RS) 돌파** | 별도 모듈 | ✗ | ✗ |

근거(코드):
- `segments()` L562: `classify(closes[: i + 1], ma_period, slope_lookback)` — 볼륨·고저 미전달.
- `_decide` 시그니처에 `channel_pos` 없음(계산은 L418, 반환은 L435).
- `_donchian` 돌파는 "종가가 직전 채널 최고 초과 + 볼륨 1.5배"만 — 박스권 자체를 구조로 안 봄.

실증(인피니트헬스케어 2026-07-10 급등봉, 일봉 MA50):
```
closes-only(밴드 경로): stage=3 breakout=none vol_sig=neutral  volatility=normal channel_pos=None
with-OHLCV(analysis):   stage=3 breakout=none vol_sig=accumulation volatility=normal channel_pos=90.2
그날 거래량 = 49일 평균의 2.9배
```
→ 배경밴드는 축적·채널상단(90.2)·2.9배 거래량을 **전혀 못 본다.** 돌파봉인데도 both 에서
breakout=none 인 건, 종가가 아직 직전 ATH를 못 넘겨서다(장중 고가·갭은 무시).

## 2. Weinstein 정통 방법론 대비 격차

Weinstein(*Secrets for Profiting in Bull and Bear Markets*)의 국면 전환 정의는 본질적으로
**"횡보 레인지(base/top) + 30주선 + 레인지 경계의 거래량 동반 돌파/이탈 + 상대강도"** 다:

- **Stage 1→2 (매수 조짐→진입)**: 바닥 다지기 레인지의 **상단 저항을 거래량 폭증과 함께
  상향 돌파**. 30주선이 평평→상승 전환. RS 라인이 0선/직전 고점 상향 돌파. → 현재 코드는
  "가격이 상승 MA 위 + clean" 으로 근사할 뿐, **레인지 상단 돌파·거래량 확인**이 밴드에 없다.
- **Stage 2 유지**: 되돌림이 30주선/직전 지지(=이전 저항이던 레벨)에서 지지받고 **HH·HL**
  갱신 지속. 거래량은 상승에 실리고 하락에 마른다. → 현재 코드는 "MA 위 + rising" 유지만 보고,
  **HH/HL 구조·되돌림 지지 확인**이 없다.
- **Stage 3→4 (분산→이탈 매도)**: 천정 레인지 **하단 지지를 거래량 동반 하향 이탈**. 30주선
  하락 전환, 가격이 그 아래. RS 하락. → 현재 코드의 "MA 아래 + falling/deep-below" 는 이탈을
  *사후에* 잡지만, **레인지 하단(넥라인)·거래량 이탈**이라는 정밀 타점이 없다.

핵심: Weinstein 은 **레인지(횡보 박스)** 를 1차 객체로 두고 그 **경계의 돌파/이탈**을 전환
신호로 쓴다. 현재 구현은 레인지를 명시적으로 만들지 않고, MA·회귀로 **간접 근사**한다.
그래서 "조짐/유지/이탈"의 정밀 타점이 흐리다.

## 3. 제안 — 시장구조를 1차 시민으로 (우선순위 순)

### P0. `segments()` 에 OHLCV 전달 (버그성, 저비용·고효과)
밴드·trend 배치가 거래량·돌파·변동성을 보게 한다. `segments(...)` 시그니처에 highs/lows/volumes 를
받아 `classify(..., volumes, highs, lows)` 로 넘기기만 하면 이미 구현된 breakout/vol/volatility 가
즉시 살아난다. **이것만으로 돌파·거래량이 밴드에 반영된다.** (analysis 배지와 밴드 불일치도 해소)

### P1. `channel_pos` 를 `_decide` 에 연결
채널 상위 20%(신고가권)면 Stage2 우위, 하위 20%(신저가권)면 Stage4 우위로 tiebreak. 이미 계산돼
있어 배선만 하면 된다.

### P2. 스윙 구조 추출 (LL/HL/LH/HH) — 신규 도메인 함수
ZigZag(임계 %, 프레임별 스케일: 일봉 5~8%, 주봉 8~12%)로 스윙 고/저 피벗을 뽑고 최근 2~3개
비교로 구조 라벨을 만든다:
- **HH+HL = 상승구조(Stage2 유지 확인)**, **LH+LL = 하락구조(Stage4 확인)**,
  **HL 첫 출현 = Stage1→2 조짐**, **LH 첫 출현 = Stage3→4 조짐**.
- 이미 `elliott.py` 에 ZigZag 피벗 로직이 있으니 재사용/공유 가능(중복 구현 회피).

### P3. 박스권(수평 지지/저항) 추출 — 신규
최근 스윙 고점들의 군집(±2%)을 저항, 저점 군집을 지지로 묶어 "박스"를 만든다. 그러면 Weinstein
정의 그대로:
- **박스 상단 + 거래량 1.5배↑ 종가 돌파 = Stage1→2 진입 타점**,
- **박스 하단 이탈 + 거래량 = Stage3→4 매도 타점**.
현재 Donchian(단일 최고/최저)보다 정밀하다(박스는 "여러 번 눌린 레벨"이라 신뢰도 높음).

### P4. 거래량 폭발/갭 이벤트
- **돌파 거래량**: 돌파봉 볼륨 / 최근 평균 ≥ 임계(예 2배) → 돌파 신뢰도 가점(현재 1.5배는 있으나
  segments 에 안 들어감 → P0 로 해결).
- **갭**: 당일 저가 > 전일 고가(상승 갭) / 당일 고가 < 전일 저가(하락 갭) → 이탈/돌파 확정 보조.
- **클라이맥스**: 변동성 확장 + 거래량 폭발 + 신고가 = Stage3(매도 클라이맥스) 조짐 강화.

### P5. 상대강도(RS)를 Stage2 확인에 결합
Mansfield RS(이미 `relative_strength.py`)가 0선/직전 고점 상향 돌파 → Stage2 진입 확신 가점,
하향 → Stage4 확신. (탑다운 점수엔 이미 RS 를 섞었으니, 국면 판별에도 확인축으로.)

### 타점 정의(제안 산출물)
판별기가 **국면 라벨**뿐 아니라 **전환 근접도/트리거**를 함께 내면 "정확한 타점"이 된다:
- `stage1_to_2_setup`: 박스 상단 근접(채널 상위) + 30주선 평탄→상승 + HL 출현 + (거래량 마름).
- `stage2_confirmed`: 박스 상단 거래량 돌파 + HH/HL 지속 + RS 상향.
- `stage2_intact`: 되돌림이 MA/이전저항에서 지지 + 구조 안 깨짐.
- `stage4_breakdown`: 박스 하단 거래량 이탈 + LH/LL + 30주선 하락 + RS 하향.

## 4. 리스크·주의

- **주봉 프레임 지연**: 돌파는 일봉에서 먼저 뜨고 주봉은 확인이 늦다. 타점은 일봉(short)에서
  잡고 주봉(mid/long)으로 국면 맥락을 대는 이원 구조가 맞다.
- **거짓 돌파(bull/bear trap)**: 거래량 미확인 돌파는 함정이 많다 → 반드시 거래량·종가(장중
  아닌) 기준 확인. 현재 하향이탈은 즉시·상향돌파는 확인 요구(비대칭)한 설계 취지와 일관.
- **과최적화**: 임계(박스 폭, ZigZag %, 거래량 배수)는 프레임별로 다르고 종목 변동성에 민감.
  기존 리뷰 루프(시각+수치 2리뷰어, 화신정공·KINX 교차검증)로 반드시 회귀 검증.
- **P2~P5 는 판별 규칙을 크게 흔든다** → 반드시 단계적으로(P0→P1 먼저 배포·검증 후 P2+) 진행.
